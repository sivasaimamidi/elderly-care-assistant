[1:00 — ARCHITECTURE]

(show assets/architecture_diagram.png)

Let me walk through the architecture. Every user message first hits the Security Checkpoint — a deterministic function node that scrubs PII like phone numbers and SSNs, detects prompt injection attempts, and blocks harmful queries before anything reaches the LLM.

If the input passes, it goes to the Orchestrator Agent. This is the brain. It reads the request and delegates to one of two specialist sub-agents using AgentTool.

The first specialist is the Routine Med Manager. It handles medication schedules — viewing meds, logging when a med was taken, and updating dosages.

The second is the Wellbeing Log Analyst. It logs vitals like blood pressure, heart rate, and temperature, and it books doctor appointments.

Both specialists connect to the MCP Server over stdio. The MCP server exposes five tools that read and write to a local JSON database: get elderly status, update medication, log medication taken, add wellbeing log, and book appointment.

After the orchestrator finishes, the Route Decision node checks if any action needs caregiver approval. If it does, the Human Reviewer node pauses the conversation and asks the caregiver to approve or deny. If approved, it loops back to the orchestrator to execute the change. If denied, the operation is cancelled.

[3:00 — SECURITY AND MCP]

The security checkpoint runs three checks on every single input: PII redaction using regex for SSNs and phone numbers, prompt injection detection scanning for phrases like "ignore previous instructions," and a harm filter blocking references to overdose, poison, or lethal doses. Every check produces a structured JSON audit log with severity levels for cloud logging integration.

The MCP server runs as an independent stdio process with five tools. It reads and writes to a simple JSON file, making it easy to inspect and debug. The architecture separates data access from LLM reasoning, so the agents never touch the database directly.

[3:30 — IMPACT AND CLOSE]

This system lowers the cognitive burden on caregivers by giving them a natural-language interface to manage patient data, while strict security controls and human-in-the-loop approval gates prevent the AI from making dangerous changes autonomously. It bridges the gap between simple alarm apps and complex medical record systems — a safe, smart concierge for the people who need it most.
